## Copywriting & Translation Expert

Expert workflow for creating and refining copy for software user interfaces, user flows, and experiences. Specializes in English to Canadian French (Québécois) translation for software applications.

### Translate English Copy to Canadian French

```
Translate this copy to Canadian French: [English copy]
```

**What it does**:
- Translates English UI/UX copy to Canadian French (Québécois)
- Uses terminology commonly found in Canadian software applications
- Maintains tone, clarity, and user experience quality
- Provides context-aware translations for software interfaces

**When to use**: Translating UI strings, error messages, button labels, tooltips, notifications, and other user-facing copy

**Detailed Steps**:

1. **Analyze the Context**
   - Identify the UI element type (button, error message, tooltip, etc.)
   - Understand the user flow context
   - Note the tone and formality level
   - Consider the target user's technical familiarity

2. **Apply Canadian French Conventions**
   - Use Québécois terminology (e.g., "courriel" not "e-mail", "clavardage" not "chat")
   - Follow Office québécois de la langue française (OQLF) guidelines for software
   - Use appropriate formality level (tu/vous) based on context
   - Maintain consistency with common Canadian software patterns

3. **Optimize for Software UX**
   - Keep translations concise for UI constraints
   - Ensure clarity in error messages and notifications
   - Use action-oriented language for buttons
   - Maintain brand voice and tone

4. **Provide Translation with Notes**
   - Primary translation
   - Alternative options if applicable
   - Context notes for ambiguous terms
   - Character length considerations if relevant

### Review and Refine Copy

```
Review this copy for clarity and tone: [copy]
```

**What it does**:
- Reviews copy for clarity, tone, and user experience
- Suggests improvements for better UX
- Ensures consistency with product voice
- Checks for accessibility and inclusivity

**When to use**: Before finalizing any user-facing copy

**Detailed Steps**:

1. **Assess Clarity**
   - Is the message clear and unambiguous?
   - Will users understand the action or information?
   - Are technical terms explained or avoided?
   - Is the language appropriate for the audience?

2. **Evaluate Tone**
   - Does it match the product's voice?
   - Is the formality level appropriate?
   - Does it feel friendly, professional, or technical as needed?
   - Is it consistent with other product copy?

3. **Check UX Best Practices**
   - Is it concise enough for UI constraints?
   - Does it guide users to the right action?
   - Are error messages helpful, not just descriptive?
   - Does it reduce cognitive load?

4. **Suggest Improvements**
   - Provide revised copy with rationale
   - Offer alternatives if applicable
   - Flag any concerns or questions

### Best Practices

**Canadian French Translation**:
- Use "courriel" for email (not "e-mail" or "mél")
- Use "clavardage" for chat (not "chat")
- Use "téléverser" for upload (not "uploader")
- Use "télécharger" for download
- Use "se connecter" for sign in (not "se logger")
- Use "créer un compte" for sign up
- Use "mot de passe" for password
- Use "nom d'utilisateur" for username
- Follow OQLF guidelines for software terminology
- Use "vous" for formal contexts, "tu" for casual/friendly products
- Maintain consistency with common Canadian software patterns

**UI Copy Best Practices**:
- Be concise: Every word should earn its place
- Be clear: Users should understand immediately
- Be helpful: Guide users to success
- Be consistent: Use the same terms throughout
- Be accessible: Use plain language, avoid jargon
- Be action-oriented: Use verbs for buttons
- Be specific: "Save changes" not "Save"

**Error Messages**:
- Explain what went wrong in plain language
- Suggest how to fix it when possible
- Avoid blaming the user
- Use consistent error message format
- Include error codes only when helpful

**Microcopy**:
- Keep it under 10 words when possible
- Focus on the essential information
- Use active voice
- Be specific, not generic
- Test with real users when possible

**Tone Guidelines**:
- Match your product's personality
- Be consistent across all touchpoints
- Consider your audience's technical level
- Balance friendliness with professionalism
- When in doubt, err on the side of clarity
